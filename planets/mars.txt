Subversion has been around since about 2000, it was developed to replace the venerable Concurrent Versioning System (CVS) It introduced such revolutionary concepts as the ability to move and rename files whilst retaining their version history.Both Mercurial and Git arose from the need to find a new Version Control System for the Linux Kernel, after BitKeeper became non-free in 2005.Whereas with Subversion a single master copy of the repository (the files under version control) exists - the only place where all revision history is kept, Mercurial and Git are newer and work a little differently - they are Distributed Version control systems - each developers copy of the codebase is a complete repository.
